# 🔱 User Stories: Neural Trace Recording [SH-1000]
## Project: Sovereign Hub v4.0
## Epic: Neural Trace Recording [SH-1000]
## Status: 📋 PLANNED | Priority: HIGH

### 🎯 Objective
Plan and implement the "Time-Machine" temporal playback features for the Sovereign Hub HUD, allowing operators to scrub through agent thought-streams, tool calls, and workspace diffs with millisecond precision.

---

### 📖 User Stories

#### 1. Neural Scrubber (The Timeline)
- **SH-1000.1: Temporal Navigation**
  - **As an operator,** I want a draggable timeline (Chrono-Scrubber) in the HUD so that I can navigate to any specific point in an agent's session.
  - **Acceptance Criteria:**
    - Timeline maps to `step_index` from the `neural_snapshots` table.
    - Dragging the needle updates the global HUD state to reflect that moment in time.
    - Visual "Heat Markers" indicate periods of high activity (bursts of tool calls/thoughts).

- **SH-1000.2: State Reconstitution**
  - **As a developer,** I want the HUD to instantly reconstruct the agent's environment (thoughts, logs, file states) when scrubbing, so I can see exactly what the agent saw.
  - **Acceptance Criteria:**
    - Thoughts and tool logs update in real-time as the scrubber moves.
    - The system handles rapid state transitions without UI lag (<100ms update).

#### 2. Thought Replay (The Narrative)
- **SH-1000.3: Thought-Stream Animator**
  - **As an auditor,** I want to replay an agent's thought-stream with a "Typewriter" effect at various speeds (1x, 2x, 4x), so I can follow its reasoning process at my own pace.
  - **Acceptance Criteria:**
    - Playback controls (Play, Pause, Speed Toggle) are integrated into the Trace tab.
    - Animation speed is adjustable on-the-fly.

- **SH-1000.4: Deep Search within Thoughts**
  - **As a researcher,** I want to search through the `payload` of recorded thoughts using keywords, so I can find the exact moment a specific decision was made.
  - **Acceptance Criteria:**
    - Search results link directly to the corresponding `step_index` on the scrubber.
    - Highlighting of search terms within the thought-stream view.

#### 3. Workspace Diffs (The Impact)
- **SH-1000.5: Visual File Diff (Ghost-Writer View)**
  - **As a code reviewer,** I want to see a side-by-side diff of file changes for every `step_index`, so I can verify the integrity of code generated by the agent.
  - **Acceptance Criteria:**
    - Uses `diff-match-patch` for line-by-line highlighting.
    - Shows "Before" and "After" states of files modified by `write` or `edit` tools.

- **SH-1000.6: Neural-Workspace Correlation**
  - **As a troubleshooter,** I want to see the specific thought that triggered a file change, so I can understand the "Why" behind a "What".
  - **Acceptance Criteria:**
    - The UI highlights the relationship between a `THOUGHT` event and a subsequent `STATE_CHANGE` or `TOOL_CALL` (file operation).

#### 4. Performance & Infrastructure
- **SH-1000.7: Latency & Storage Monitoring**
  - **As a system admin,** I want to track the performance overhead of the recording layer, so I can ensure it doesn't degrade agent responsiveness.
  - **Acceptance Criteria:**
    - `recording_latency_ms` is tracked and visible in the Analytics tab.
    - Disk usage metrics for the `neural_snapshots` table are graphed.
    - Overhead remains below 50ms per event.

### 🗺️ 30/60/90 Day Action Plan

- **30 Days:** Infrastructure & Core Recording. Complete SQL migrations, implement `TraceRecorder.js`, and begin background data collection.
- **60 Days:** Playback UI (Alpha). Functional Chrono-Scrubber and basic Thought Replay. Integration with SH-700 for latency monitoring.
- **90 Days:** Full "Time-Machine" Experience. Visual Diffs (Ghost-Writer), Deep Search, and optimized multi-session playback.

### 💰 Monetization Ideas
1.  **"Agent Audit" Enterprise Tier:** Charge for long-term retention of neural traces (e.g., 1 year vs 30 days) for compliance and legal safety.
2.  **Training Data Export:** Allow users to export high-quality "Thought -> Action" traces to fine-tune their own local models.
3.  **Collaborative Debugging:** A hosted "Trace Cloud" where developers can share a snapshot link to a specific failure point for community help.
