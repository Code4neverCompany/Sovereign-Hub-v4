### 1. OVERVIEW
The "Node Genesis" layer is the biological engine of the Sovereign Hub, enabling it to autonomously expand its physical footprint across cloud providers. It monitors the swarm's cognitive load and hardware utilization, deciding when to "bud" a new node, provision it, and securely integrate it into the existing Tailscale mesh.

---

### 2. INFRASTRUCTURE PROVIDERS: Multi-Cloud Integration
To avoid vendor lock-in and optimize for cost/performance (especially for GPU tasks), the Hub utilizes a provider-agnostic interface.

**Target Providers:**
- **AWS/GCP:** General purpose compute, storage, and reliability.
- **RunPod / Lambda Labs:** High-performance, low-cost GPU instances for deep research and model training.
- **DigitalOcean:** Lightweight, fast-provisioning nodes for edge tasks.

**Implementation:** `InfraManager.js`
- Uses provider SDKs (e.g., `@aws-sdk/client-ec2`, `google-cloud/compute`).
- Maintains a registry of "Machine Images" (AMIs/Snapshots) pre-configured with the base Sovereign OS environment.

### 3. GENESIS LOGIC: `NodeGenesis.js` Middleware
Location: `/home/skywork/workspace/PROJECTS/sovereign-hub/v4/core/NodeGenesis.js`

This logic acts as the "Pituitary Gland" of the swarm, regulating growth.

**Decision Engine:**
1. **Heartbeat Monitoring:** Subscribes to `SwarmPulse.js` for real-time telemetry.
2. **Saturation Triggers:**
   - **Scale UP:** Average CPU > 75% OR Memory > 80% OR Queue Latency > 2s for a sustained 5-minute window.
   - **Scale DOWN:** Average utilization < 20% for a 30-minute window (Graceful Draining).
3. **Provisioning Request:** Dispatches a `PROVISION_NODE` event to the `InfraManager`.

### 4. REPLICATION PROTOCOL: Autonomous Identity & Sync
When a new instance boots, it must transition from "Generic Cloud Instance" to "Sovereign Node" without manual intervention.

**The "Genesis Boot" Sequence:**
1. **Cloud-Init:** The `InfraManager` injects a "Genesis Script" via UserData.
2. **Environment Cloning:** The script pulls the latest stable branch of the Sovereign repository.
3. **Tailscale Identity:** 
   - Uses a pre-generated, one-time `TS_AUTH_KEY` to join the private mesh.
   - Assigns a unique hostname: `sh-node-[region]-[id]`.
4. **Credential Injection:**
   - New node requests its environment secrets (API Keys, DB Strings) from the Orchestrator using a temporary **Genesis Key**.
   - Orchestrator verifies the node's fingerprint (Cloud Metadata) before releasing secrets.
5. **Registration:** Node announces its `READY` state to Supabase.

### 6. SECURITY: Genesis Key & Rotation Logic
Ensures that only authorized nodes can join the swarm.

1. **Genesis Key:** A 256-bit cryptographically secure token generated by the Orchestrator.
2. **One-Time Use:** Each key is valid for a single node registration and expires after 10 minutes.
3. **Rotation:** 
   - Orchestrator rotates the master registration secret every 24 hours.
   - Active nodes use mTLS (Mutual TLS) with certificates issued during the Genesis Boot for all inter-node communication.
4. **Kill Switch:** If a node is compromised, the Orchestrator revokes its Tailscale identity and database access tokens instantly.

### 7. DATA SCHEMA (SQL Implementation)

```sql
-- Track Physical Infrastructure Nodes
CREATE TABLE IF NOT EXISTS infrastructure_nodes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    node_name TEXT UNIQUE NOT NULL,
    provider TEXT NOT NULL, -- 'AWS', 'GCP', 'RUNPOD', etc.
    instance_id TEXT UNIQUE,
    region TEXT,
    ip_internal TEXT, -- Tailscale IP
    status TEXT DEFAULT 'PROVISIONING' CHECK (status IN ('PROVISIONING', 'BOOTING', 'ONLINE', 'DRAINING', 'OFFLINE')),
    instance_type TEXT,
    hourly_cost NUMERIC(10, 4),
    last_heartbeat TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Track Scaling Events for Audit & Optimization
CREATE TABLE IF NOT EXISTS scaling_events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    event_type TEXT CHECK (event_type IN ('SCALE_UP', 'SCALE_DOWN')),
    reason TEXT, -- e.g., 'High Latency', 'Budget Adjustment'
    node_id UUID REFERENCES infrastructure_nodes(id),
    cost_impact NUMERIC(10, 4),
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Genesis Keys (Short-lived)
CREATE TABLE IF NOT EXISTS genesis_keys (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    key_hash TEXT NOT NULL,
    is_used BOOLEAN DEFAULT false,
    expires_at TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ DEFAULT now()
);
```
