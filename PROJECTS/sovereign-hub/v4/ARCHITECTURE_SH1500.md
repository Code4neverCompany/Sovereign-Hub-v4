## 2. DATA ARCHITECTURE: CONTENT ENGINE
The `marketing_content` table serves as the primary repository for all AI-generated social and promotional assets.

### Supabase Table: `marketing_content`
```sql
CREATE TABLE IF NOT EXISTS marketing_content (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    product_id UUID REFERENCES product_blueprints(id) ON DELETE CASCADE,
    
    -- Content Details
    content_type TEXT NOT NULL, -- 'X_POST', 'LI_ARTICLE', 'BLOG_HOOK', 'AD_COPY'
    status TEXT DEFAULT 'DRAFT', -- 'DRAFT', 'OPTIMIZING', 'READY', 'POSTED', 'ARCHIVED'
    body_text TEXT NOT NULL,
    headline TEXT,
    
    -- Viral Metadata
    viral_hook_id UUID, -- Reference to the specific psychological hook used
    target_keywords TEXT[], -- Keywords intended for the Signal Boost mechanism
    media_assets JSONB DEFAULT '[]'::jsonb, -- { type: 'IMAGE', url: '...', prompt: '...' }
    
    -- Performance Tracking
    scheduled_at TIMESTAMPTZ,
    posted_at TIMESTAMPTZ,
    performance_score NUMERIC DEFAULT 0, -- Calculated Viral Momentum
    
    -- Agent Metadata
    creator_agent TEXT DEFAULT 'ALEX', -- The agent that initiated the draft
    optimizer_agent TEXT DEFAULT 'JORDAN', -- The agent that refined the hook
    visualizer_agent TEXT DEFAULT 'MAYA', -- The agent that generated the assets
    
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);

-- Indexing for rapid retrieval
CREATE INDEX IF NOT EXISTS idx_marketing_product ON marketing_content(product_id);
CREATE INDEX IF NOT EXISTS idx_marketing_status ON marketing_content(status);
```

## 3. VIRAL PROTOCOL: SIGNAL BOOST MECHANISM
The **Signal Boost** protocol is a reactive monitoring system that bridges internal product value with external trending conversations.

### Workflow:
1.  **Trend Monitoring:** A background worker (Forge Scout) polls X/Twitter and LinkedIn for keywords defined in `product_blueprints.metadata.tags`.
2.  **Signal Injection:** When a high-velocity keyword match occurs, the system triggers Jordan [Strategist] to select a "Ready" post from `marketing_content`.
3.  **Contextual Hooking:** Jordan rewrites the "Intro Hook" of the post to reference the trending topic/thread directly.
4.  **Autonomous Deployment:** The modified post is pushed to the respective platform via the Hub's social integrations (or logged for manual approval in the HUD).

## 4. UI SPECIFICATION: GROWTH HUD
The 'Growth' tab in Sovereign Hub v4.0 provides a high-fidelity view of the Hive's external reach.

### Key Components:
-   **Social Reach Matrix:** Aggregate view of Impressions vs. Conversions across all active prototypes.
-   **Content Backlog:** A kanban-style view of the `marketing_content` table (Draft -> Optimizing -> Ready).
-   **Viral Momentum Score (VMS):** A proprietary metric calculated as:
    `VMS = (Engagement Velocity * Conversion Rate) / Time Since Post`
-   **Signal Map:** A word cloud of "Hot Keywords" the Hive is currently tracking for Signal Boost injection.
